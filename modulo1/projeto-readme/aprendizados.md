Para iniciar nossos estudos, começamos aprendendo sobre o **Terminal**, a aula foi muito explicativa, mas a dinâmica foi essencial para colocar em prática o que aprendemos. No exercício tínhamos que desvendar um crime através do terminal, começamos por uma pista e precisávamos navegar entre as pastas e buscar entre os arquivos por pistas. Foi desafiador e super divertido, me senti um verdeiro detetivo e agora estou afiadíssimo com o terminal agora.

Comandos aprendidos:

    cd .. - volta uma pasta
    cd pasta - acessa a pasta solicitada
    mkdir pasta - cria uma pasta chamada pasta
    touch arquivo.ext - cria um arquivo com a extensão escolhida
    rm arquivo - remove um arquivo criado (cuidado)
    pwd - exibe o local que você se encontra no diretório
    grep palavra arquivo - procura uma certa palavra em um determinado arquivo
    ls - exive todos os arquivos presentes naquela pasta
    cat arquivo - exibe a informação de texto do arquivo na tela do terminal
    head arquivo - imprime as 10 primeiras linhas de um arquivo na tela do terminal
    tail arquivo - imprime as 10 ultimas linhas de um arquivo na tela do terminal

Existem muitos outros, mas esses são os principais.

## Git e GitHub

E depois disso aprendemos sobre **Git e GitHub** e por mais que eu tenha feito aulas ontem, pela primeira vez consegui entender planemente como se navega entre as branchs, como elas funcionam. Pra completar o exercício é super explicativo, e permite que você utilize bastante dos seus conhecimentos aprendidos na prática.

Estou muito contente com os resultados e como estou colocando em prática na Boticário, então sinto que estou crescendo muito rápido e fico super animado com isso. Estou doido para aprender ainda mais e ser um super dev da Boticário formado pela Labenu.

Resumidamente, git é um sistema de versionamento, que permite você gerenciar versões do seu codigo. É muito útil principalmente quando temos muitas pessoas trabalhando no mesmo código, pois assim podemos prevenir problemas.

O GitHub é um local de repositórios remotos, onde você pode colocar e gerenciar seus códigos online.

Alguns comandos aprendidos:

    git init - iniciar um repositório na sua maquina
    git clone url - clona repositório remoto na sua maquina 
    git status - verifica o status de versionamento
    git add - adiciona arquivos para serem comitados
    git commit -m "mensagem" - faz um commit com uma mensagem resumindo o que foi feito no commit
    git push origin branch - sobe para o repositório remoto as alterações feitas na sua branch
    git branch nome-da-branch - cria uma ramifição do projeto
    git checkout nome-da-branch - alterna entre as branchs
    git pull - atualiza o repositório local com a mais recente versão da branch

Existem muitos outros, mas esses são um dos mais importantes.
